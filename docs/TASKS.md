## Phase 3 — Async & resumability
The resumable state machine already exists (Phase 2): the run loop *suspends* via
`Outcome{Finished,HostCall,EpochDeadline,Grow}` and the generic driver
`exec::host::execute<T>` services suspends in a `loop`. Phase 3 turns that driver
into a `Future` and adds async-host + yield suspend reasons. **No fibers / native
stack switching** — we're a state machine (ARCHITECTURE §2.4), so suspend/resume is
just parking the `Execution` struct; `Config::async_stack_size` is therefore a
documented no-op kept only for wasmtime API parity. Note there is **no
`Store::new_async`** in wasmtime — the `Store` constructor is unchanged; async-ness
comes from `Config::async_support` on the `Engine`. The async "constructors along the
way" are the *instantiation* and *allocation* paths (`Instance::new_async`,
`Linker::instantiate_async`, `Memory::new_async`, `Table::new_async`) — they exist
because the `start` function is guest code that can suspend and because the resource
limiter can be async.
- [x] **#25a** Async foundation — `async` cargo feature is real: `Config::async_support` sets a flag snapshotted onto `EngineInner` (`Engine::is_async()`); `Config::async_stack_size` is a documented no-op (state machine — no native fiber stack, ARCHITECTURE §2.4). Sync `Func::call` now rejects an async store (`use call_async`). Parkability codified: `Execution` is self-contained (owns its stacks, no borrow into the `Store` held across an `Outcome` suspend) with a compile-time `Send` assertion guarding it. Tested under `--features async`.
- [x] **#25b** Async core driver + `call_async` + async instantiation — `exec::host::execute_async` mirrors the sync driver loop as a `Future` (reuses the sync servicing helpers; `.await` points arrive in #25c/#25d). `Func::call_async`/`TypedFunc::call_async` + `Instance::new_async` + `Linker::instantiate_async` (the `start` fn runs through `execute_async`, so it can suspend). Faithful to wasmtime: `impl AsContextMut` with **no `T: Send` bound** → conditionally-`Send` future; sync `new`/`call` reject async stores, async entries require `async_support`. `resolve_callee` extracted (shared by `call`/`call_async`); `instantiate_resolve_start` shared by `new`/`new_async`. 8 async unit tests (`src/func/async_tests.rs`, `pollster::block_on`).
- [x] **#25c** Async host functions — `Func::new_async`/`wrap_async` + `Linker::func_new_async`/`func_wrap_async` returning boxed futures (`src/func/async_func.rs`; typed path needs no arity macro — `wrap_async` takes one `P: WasmResults` tuple → `into_async_func` helper in `into_func.rs`). New `FuncEntity::HostAsync` + `Store.async_host_funcs`; the run loop suspends `Outcome::HostAsync` (via `DoHostAsyncCall`, incl. `call_indirect`), and `execute_async` awaits the future (`invoke_host_async`) before resuming — `execute_async` is now genuinely suspending (the `unused_async` allow is gone). Faithful to wasmtime: `for<'a> Fn(Caller, …) -> Box<dyn Future + Send + 'a>`, `T: Send + 'static`; conditionally-`Send` future. Sync driver + sync `Func::call` reject async host fns. 6 async-host unit tests with a `YieldOnce` future proving real park/resume.
- [x] **#25d** Fuel & epoch async yield — yield to the executor instead of trapping. Fuel modeled as an active/reserve split on `StoreInner` (`consume_fuel_step` → `FuelStep::{Ran,NeedYield,Exhausted}`, `refuel_from_reserve`); no interval ⇒ reserve 0 ⇒ accounting identical to #21 (determinism preserved). `Store::fuel_async_yield_interval` (requires `consume_fuel` + async, rejects 0) and `epoch_deadline_async_yield_and_update`. ## Phase 5 — GC
**Sequencing: allocate-only (`Collector::Null`) first, real collector last.** The whole
GC *surface* — heap layout + value model (#27a), type system (#27d), cross-module identity
(#27c), host API (#27b), aggregate instructions (#27e), casts/eq/extern (#27f) — is built
and made spec-testable against a **null collector** that allocates and **never reclaims**
(heap exhaustion → limiter/`Trap`). This matches `wasmtime`'s `Collector::Null` and lets
everything downstream land + pass conformance without a tracing GC in the way. The real
non-moving mark-sweep collector (#27g) is then dropped in **at the end of the phase** as a
behind-the-scenes upgrade — no surface change. The lettered subtasks are grouped by
concern, not strict execution order, but #27g deliberately comes after the feature set.
- [x] **#27a** GC heap + value model (**null collector**) — **pure foundation** (its callers, host API #27b + instructions #27e, are out of scope, so the new surface is `#[allow(dead_code)]` until they land; unit-tested only; `WasmFeatures::GC` stays off ⇒ no spec movement by design). New `src/store/gc.rs`: a grow-only handle-table `GcHeap` of `GcObject { header: GcHeader, body: Struct(Vec<Val>)|Array(Vec<Val>) }` on `StoreInner` (mirrors the externref arena; freed on `Store` drop), `alloc`/`get`/`get_mut` delegators (`alloc_gc`/`gc_object`/`gc_object_mut`). Header carries `type_index` (module-placeholder; engine-canonical id is #27c) + **inert** `mark`/`generation` (live at #27g). `i31` **unboxed in the handle**: top-bit-tagged `anyref` handle encoding (`anyref_handle_i31`/`_slot` + `decode_anyref_handle` → `AnyRefHandle::{I31,Slot}`, sign-extending). Allocate-only; heap exhaustion → `Trap::AllocationTooLarge` (limiter-batch budget is #31). `Config::collector`/`gc_memory_threshold` already existed as knobs → wired onto `EngineInner` (`collector` recorded for #27g; `gc_memory_threshold` sizes each store's `GcHeap` ceiling for now, default 1 GiB). `Val`/`ref.null`/defaultability already cover the GC ref set under `Val::AnyRef` + the `none`/`nofunc`/`noextern` bottoms (verified, no change).
